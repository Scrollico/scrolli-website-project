**Step 1: Compile the TypeScript plugin to CommonJS JavaScript.**

The existing plugin at `scripts/eslint-plugin-scrolli-design.ts` uses ESM `export` syntax and TypeScript types. ESLint requires CommonJS plugins. Create `scripts/eslint-plugin-scrolli-design.js` as the compiled CJS version:

- Convert all `export const` to `module.exports` pattern
- Remove TypeScript type annotations (`Rule.RuleModule`, `: any`)
- Remove the `import type { Rule } from 'eslint'` import
- Keep ALL regex patterns and rule logic identical
- Structure the module.exports as:
```javascript
/** @type {import('eslint').ESLint.Plugin} */
module.exports = {
  rules: {
    'no-hardcoded-design-values': { /* ... rule from noHardcodedDesignValues ... */ },
    'require-typography-components': { /* ... rule from requireTypographyComponents ... */ },
  },
};
```

**Step 2: Create a Node.js setup script for plugin registration.**

Create `scripts/setup-eslint-plugin.js` (NOT a shell echo command — avoids nested quoting issues in CI/CD):

```javascript
#!/usr/bin/env node
/**
 * Registers the local ESLint plugin so ESLint can resolve "scrolli-design".
 * Run via: node scripts/setup-eslint-plugin.js
 * Called automatically by npm postinstall.
 */
const fs = require("fs");
const path = require("path");

const pluginDir = path.join(__dirname, "..", "node_modules", "eslint-plugin-scrolli-design");

fs.mkdirSync(pluginDir, { recursive: true });

fs.writeFileSync(
  path.join(pluginDir, "index.js"),
  'module.exports = require("../../scripts/eslint-plugin-scrolli-design.js");\n'
);

fs.writeFileSync(
  path.join(pluginDir, "package.json"),
  JSON.stringify({ name: "eslint-plugin-scrolli-design", version: "1.0.0", main: "index.js" }, null, 2) + "\n"
);

console.log("eslint-plugin-scrolli-design registered in node_modules");
```

**Step 3: Add postinstall script to package.json.**

In `package.json` under `"scripts"`, add:
```json
"postinstall": "node scripts/setup-eslint-plugin.js"
```

If a `postinstall` script already exists, chain with `&&`.

**Step 4: Run the setup script immediately:**
```bash
node scripts/setup-eslint-plugin.js
```

**Step 5: Wire plugin into `.eslintrc.json`.**

Update `.eslintrc.json` to:
```json
{
  "extends": "next/core-web-vitals",
  "plugins": ["scrolli-design"],
  "rules": {
    "react/no-unescaped-entities": 0,
    "@next/next/no-img-element": "off",
    "scrolli-design/no-hardcoded-design-values": "warn"
  }
}
```

Note: No `settings.resolvePluginsRelativeTo` needed — the setup script handles resolution via node_modules.

**Step 6: Set rule to `"warn"` not `"error"`** so existing codebase doesn't fail CI during migration. After migration plans complete, it can be upgraded to `"error"`.
