       ScrollTrigger `pin` automatically unpins when scroll passes the `end` trigger point. Verify:
       - The outer `<section>` has `height` set to `SCROLL_DISTANCE` (e.g., "300vh") — this creates the scroll runway
       - ScrollTrigger `end` is set to `+=300vh` (matching section height)
       - No `position: fixed` or manual positioning is applied that could override ScrollTrigger's pin behavior
       - The section wrapper in `pricing/page.tsx` has `className="w-full relative"` (no z-index that blocks unpinning)

    3. **Performance optimizations for 60fps (SCROLL-03):**
       Add `will-change: transform` to card elements for GPU compositing:
       ```typescript
       // In the card JSX, add to the style prop:
       style={{
         width: CARD_WIDTH,
         height: CARD_HEIGHT,
         willChange: "transform",
       }}
       ```

       Ensure GSAP is using `transform` (not `top`/`left`) for card positioning — GSAP defaults to transform when using `x`, `y`, `rotation`, `scale` properties (which it does). Verify no `top:`, `left:`, or `margin` properties are animated.

       Add `force3D: true` to the ScrollTrigger GSAP tweens for GPU acceleration:
       ```typescript

         force3D: true,
         // ...
       }, position);
       ```

    4. **Cleanup robustness:**
       Ensure the cleanup function in useEffect kills ScrollTrigger instances scoped to this component, not globally:
       ```typescript
       return () => {
         // Kill only our ScrollTrigger, not all on the page
         st.kill(); // where st is the ScrollTrigger.create() return value
         tl.kill();
       };
       ```
       Store the ScrollTrigger.create() return value in a variable `const st = ScrollTrigger.create({...})` and kill that specific instance.

    5. **Resize handling:**
